WealthSignal · Local Dev Stack

All the links, one page.

Every URL the local stack exposes — portals, FastAPI services, Airflow, MLflow, Postgres, Kafka, the whole observability chain. Click any card to open. Everything assumes npm run run:local:all is up.

⚠ All services bind to localhost. If a card leads to ERR_CONNECTION_REFUSED, the specific container probably isn't up — run npm run run:local:status to see which tier is down.
✨

Suggested next try

Three steps, in order, to see the whole stack working end-to-end.
Bootstrap MLflow experiments

Runs the bootstrap_mlflow_experiments Airflow DAG via admin_api — creates the four production (wealth-*) and four mastery (mastery/wealth-*) experiment slots.

npm run setup:local:mlflow:experiments

Then refresh MLflow UI and you should see the eight experiments listed.

Airflow → MLflow
Load the product catalogue

From Admin Portal left nav → Domain Services → Product Catalog → Initial Data Load Setup. Click Start Initial Data Load.

The page polls every 2 s. On success, shows inserted / updated / total counts — load_product_catalog DAG runs the UPSERT into product_catalog.products.

Portal → Airflow → Postgres
Try natural-language search

Open the Customer Portal. Type "tax-efficient ISA for first-time investor" into the hero search box and hit Search.

Top-6 ranked matches come back from product_catalog_api's pg_trgm NLP endpoint, each card showing a match score + matched-term chips.

Portal → API → Postgres (pg_trgm)
