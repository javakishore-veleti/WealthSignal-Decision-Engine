WealthSignal · Decision Engine

NPM Commands Reference

Every orchestration command in this repo lives in the root package.json. This guide groups them by when you use them so a new contributor can get productive in minutes.

⚠ Activation has to run in your own shell — npm can't modify the parent shell's PATH. Whenever a command begins with source …, copy-paste and run it yourself.
🆕

First-time setup

Run once per machine.

One command bootstraps everything: conda env, package scaffold, Python deps, portal deps, pre-commit hooks, and a next-step cheat-sheet banner.

CommandWhat it does
npm run setup:local:allFull bootstrap. Chains all the steps below in order.
npm run setup:local:condaCreates (or updates) the conda env at $HOME/runtime_data/python_venvs/WealthSignal-Decision-Engine. Idempotent.
npm run setup:local:scaffoldCreates engine/wealthsignal/{config,data,models,training,serving,utils,tracking}/__init__.py. Never overwrites.
npm run setup:local:pythonpip install -e .[dev,db,seed] inside the conda env.
npm run setup:local:portalsInstalls npm workspace dependencies for admin_portal + customer_portal.
npm run setup:local:precommitWires pre-commit hooks for ruff + black.
npm run setup:local:airflowRuns the Airflow DB migration + admin-user bootstrap (needs Postgres up).
npm run setup:local:postPrints the final banner with next-step commands.
Immediately after setup: 
# Activate the conda env in THIS shell:
source DevOps/Local/python/activate.sh

Daily driver commands

Things you run every session.
CommandWhat it does

Typical workflows

Copy-paste recipes for common situations.
Brand-new machine
git clone git@github.com:javakishore-veleti/WealthSignal-Decision-Engine.git
cd WealthSignal-Decision-Engine

npm run setup:local:all                   # full bootstrap (~5-10 min)
source DevOps/Local/python/activate.sh    # activate in this shell
npm run run:local:all                     # start every tier
Starting a working session
source DevOps/Local/python/activate.sh
npm run run:local:all
# … work …
npm run run:local:shutdown
source DevOps/Local/python/deactivate.sh
Iterating on a single middleware service
npm run run:local:docker:postgres             # only what the service needs
npm run run:local:middleware:product-catalog  # foreground reload on file change
Resetting a broken env
npm run env:recreate                          # remove + recreate + pip-install

Pointers: root package.json is operational only (no runtime JS). Angular portals live under portals/ as npm workspaces. Conda prefix overridable via $WEALTHSIGNAL_CONDA_PREFIX. PIDs + logs for background processes live under .local/ (gitignored). DB migrations are Alembic, triggered manually via the Database Migration GitHub Actions workflow.
